# 저층부 형태 시뮬레이터 v1

도시 설계 가이드라인 파라미터를 슬라이더로 조작하면, 3D 건물 매스가 실시간으로 반영되고 공개성·보행성 등 6개 축으로 평가 점수를 산출하는 **도시 저층부 형태 시뮬레이션 리서치 프로토타입**입니다.

---

## 프로젝트 목적

도시 설계 가이드라인(지구단위계획, 건축 조례 등)이 건물의 저층부 형태와 가로 활성도에 어떤 영향을 미치는지 직관적으로 시뮬레이션하고 비교하기 위한 도구입니다. 서울·도쿄 마루노우치·함부르크 하펜시티의 실제 가이드라인 수치를 프리셋으로 제공하고, 슬라이더 조작만으로 도시 간 형태 차이를 즉시 비교할 수 있습니다.

## 화면 구성

```
헤더: P-index / D-index 뱃지 | 대지 선택 | 보행자 토글 | 광화문 지도 토글 | 건축가 스타일 | 비교모드 | CSV/PNG 내보내기
├─ 왼쪽: ParameterPanel
│   ├─ 도시 프리셋 버튼 (서울 / 마루노우치 / 하펜시티 / 혼합)

└─ 오른쪽
    ├─ 3D 뷰어 (BuildingMass)
    └─ 하단 차트 (레이더 / 산점도 / 점수 바)
```

## 3D 뷰어 모드

### 1. 기본 직사각형 모드
- 50m × 80m 가상 대지 기준
- 저층부(파란색) + 상층부(회청색) + 공개공지(녹색) + 캐노피 + 필로티 렌더링
- 대지 경계선(황색), 층수/높이 텍스트 표시

### 2. 광화문 지도 모드
- 로컬 `gwanghwamun.geojson`에서 실제 건물 footprint 로드
- Overpass API로 실제 도로 데이터 로드 (localStorage 캐시 적용)
- 모든 필지에 현재 파라미터 즉시 적용
- 지도 타일 배경 ON/OFF 전환 가능 (Carto 타일)

### 3. 커스텀 대지 모드 (VWorld 연동)
- 좌측 상단 **대지 선택** 버튼으로 진입
- VWorld 지적도 위에서 실제 필지 클릭 선택 (단일/다중/다각형)
- 주소 검색으로 지도 이동
- 선택한 필지의 실제 형태(WGS84 폴리곤)를 3D 씬에 가져와 파라미터 적용
- Overpass API로 주변 도로·건물 데이터 자동 로드
- 선택 필지 경계선(황색)·주변 건물·도로 함께 렌더링

## 파라미터 구조 (16개)

| 그룹 | 파라미터 | 단위 |
|------|---------|------|
| **법적 한도** | 건폐율, 용적률, 높이 제한 | %, %, m |
| **이격 / 공지** | 전면 후퇴, 측면 이격, 후면 이격, 공개공지 비율, 공개공지 최소폭 | m, m, m, %, m |
| **일조사선** | 기준 높이, 사선 비율 | m, :1 |
| **저층부 형태** | 저층부 층고, 입면 투명도, 1층 전면 활성 비율, 캐노피 깊이, 캐노피 연속성, 필로티 비율, 필로티 깊이, 공공보행통로 폭 | m, %, %, m, %, %, m, m |
| **가로 활성도** | 출입구 빈도, 비활성 전면 비율, 야간 활성화 지수 | 개/100m, %, 0-1 |

## 도시 프리셋 4개

| 도시 | 특징 |
|------|------|
| **서울 (현행)** | 상업지역 건폐율 60%, 중·고층 위주, 후퇴 최소 |
| **도쿄 마루노우치** | 입면 투명도·상업 연속성 높음, 가로 활성도 중시 |
| **함부르크 하펜시티** | 공개공지·전면 후퇴 넉넉, 필로티·캐노피 강조 |
| **혼합** | 세 도시 중간값 혼합 |

---

## 점수 계산 시스템 (6개 축, 0~100점)

| 축 | 계산식 |
|----|--------|
| **공개성 (Publicness)** | 공개공지 × 0.45 + 캐노피연속성 × 0.30 + 필로티 × 0.25 |
| **보행성 (Walkability)** | 보행통로폭 × 0.35 + 출입구빈도 × 0.40 + 캐노피깊이 × 0.25 |
| **전면공간 (Setback)** | 전면 후퇴 거리 정규화 |
| **체류성 (Dwellability)** | 공개성 × 0.30 + 전면공간 × 0.30 + 야간활성 × 0.40 |
| **입면개방성 (Facade Openness)** | 투명도 × 0.35 + 상업연속성 × 0.35 + (100-비활성전면) × 0.30 |
| **사업성 (Developability)** | 용적률 정규화 |

- **P-index**: 공공성 5개 축 평균 (공개성·보행성·전면공간·체류성·입면개방성)
- **D-index**: 사업성 단독

## 건축가 스타일 2종

슬라이더 파라미터와 독립적으로 3D 렌더링 스타일을 적용합니다.

| 스타일 | 특징 |
|--------|------|
| **안도 다다오** | 노출 콘크리트 단일 매스, 거푸집 수평 줄눈, 전면 빛의 십자 슬릿, 강한 사광 조명 |
| **자하 하디드** | 파라메트릭 전단(shear) 매스 28슬라이스, 캔틸레버 슬래브 플레이트, 알루미늄 패널 재질 |

두 스타일 모두 저층부(1-3층 / ~12m)는 파란색(`#2563EB`), 상층부는 각자의 스타일 색상 적용.

## 보행자 시뮬레이션

- 최대 50명 인스턴싱 에이전트가 대지 주변을 실시간으로 이동
- 파라미터 점수와 직접 연동:
  - 활성 에이전트 수 ← 공개성 점수
  - 이동 속도 ← 보행성 점수
  - 정지 확률/시간 ← 체류성 점수
  - 건물 입면 쪽 이동 경향 ← 입면개방성 점수

## 주요 데이터 흐름

```
사용자 슬라이더 조작
  → params 상태 변경 (App.jsx)
  → calculateScores(params) 재계산
  → calculateMass(params, lot) 재계산
  → 3D 매스 즉시 갱신 (BuildingMass)
  → 레이더·산점도·점수바 갱신
  → 보행자 에이전트 행동 변화 (실시간)

커스텀 대지 선택
  → VWorld Data API로 필지 폴리곤 조회
  → 좌표 로컬 미터 변환 (WGS84 → 씬 좌표)
  → Overpass API로 주변 도로·건물 로드 (캐시 우선)
  → CustomParcelScene 렌더링
```

## 알려진 한계 및 미완성 항목

- `rulesets.js` 수치는 추정치 — 실제 조례·가이드라인 원문 검증 필요
- Overpass API 타임아웃 빈번 (localStorage 캐시로 임시 대응)
- 기본 대지 크기 50×80m 고정 — 대지 크기 변경 시 scoring 정규화 범위 재검토 필요
- 건축가 스타일은 커스텀 대지 모드·광화문 모드에서 일부 요소 미반영
- 일조사선·측면/후면 이격은 파라미터만 존재, 3D 형태 반영 미완
